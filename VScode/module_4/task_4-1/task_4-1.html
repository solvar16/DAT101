    2. [it is a way of programming that makes it easier maintaining a well structured and reusable code. It will allow you to use a specific set of code several times without having to write it new every time].

  2. [In Task 1, we create a static object called "AccountType." Explain what this object is used for and how this type of object can help us in software development.]

    3. [static object is used to store predefined account types, like "Saving" or "Pension." It helps standardize values, reducing errors and making the code easier to read and maintain. Using such objects in software development promotes consistency and simplifies updates across the program.]

  3. [Discuss why we use encapsulation in classes. Why do we want to "hide" code within our class? Explain how we can create "private" and "public" members in a constructor function.]

    4. [We use encapsulation to protect the inner workings of a class, allowing us to control access to its data and methods. "hiding" prevents other parts of the code from directly changing important data, it reduces errors and improves maintainability. We create private members using symbols like _ (by convention) or with #. Public members are accessible directly. This approach keeps sensitive data private while exposing only what is necessary.]

  4. [Explain what happens when we use the "new" command in JavaScript.]

    5. [When we use the new command, it creates a new instance of an object based on a constructor function. It creates a new empty object, sets the this keyword in the constructor to refer to that new object. It links the new object to the prototype of the constructor, allowing it to inherit properties and methods. It also returns the new object, making it an instance of the constructor.]
